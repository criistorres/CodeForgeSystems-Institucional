Segurança

Introdução

O ASP.NET Core Identity é uma biblioteca da Microsoft para gerenciamento de autenticação e autorização em aplicações ASP.NET Core. Ele fornece recursos como login, gerenciamento de usuários, funções, políticas de segurança, tokens e integração com autenticação externa.

JJInfinity adota essas práticas e também faz uso extensivo de recursos prontos do ASP.NET Core, além de utilizar o YARP (Yet Another Reverse Proxy) para roteamento avançado e balanceamento de carga em cenários distribuídos.

As configurações abaixo representam as políticas de segurança padrões do JJInfinity.

Menor Privilégio

Usuários autenticados não possuem acesso a nenhuma funcionalidade por padrão. O acesso deve ser explicitamente concedido via roles, claims ou policies.

Senhas Irreversíveis (Hashing)

Senhas são armazenadas utilizando algoritmos de hash criptograficamente seguros com salt. Não é possível reverter o hash para descobrir a senha original.

Bloqueio de Conta (Lockout)

Contas são bloqueadas automaticamente após múltiplas tentativas de login mal-sucedidas consecutivas, protegendo contra ataques de força bruta.

Autenticação em Duas Etapas (2FA)

Suporte nativo a autenticação em duas etapas usando e-mail, SMS ou aplicativo autenticador.

Autorização Baseada em Roles e Claims

Controle granular de permissões via atribuição de roles e claims ao usuário. As views e APIs verificam autorização antes de executar qualquer ação.

Validação de Anti-Forgery Tokens

Todas requisições POST que modificam dados exigem tokens antifalsificação, prevenindo ataques CSRF.

Cookies Seguros

Cookies de autenticação usam flags HttpOnly, Secure e SameSite=Strict por padrão.

Proteção de Dados (Data Protection API)

Tokens de recuperação, autenticação e redefinição de senha são criptografados com a Data Protection API.

Proteção contra Enumeração de Usuários

Mensagens de erro genéricas são utilizadas em autenticação para evitar que invasores confirmem a existência de usuários por tentativa e erro.

Proxy Reverso com Isolamento de Módulos

O YARP (Yet Another Reverse Proxy) é utilizado para expor apenas endpoints públicos da aplicação. Módulos internos e administrativos não são acessíveis diretamente pela internet, garantindo isolamento de funcionalidades sensíveis.

Rate Limiting

Mecanismos de limitação de requisições são aplicados a endpoints críticos para prevenir abusos e ataques de negação de serviço (DoS). O sistema pode impor limites por IP, usuário ou rota específica.

Expiração e Renovação de Tokens

Tokens de autenticação têm tempo de expiração controlado.
