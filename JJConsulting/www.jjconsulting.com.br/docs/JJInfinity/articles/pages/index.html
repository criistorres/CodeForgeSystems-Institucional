JJPages é o módulo do Infinity utilizado para criação de páginas customizadas.

Cabeçalho

Dados básicos da página, que são o nome que identifica ela e sua conexão SQL. A conexão SQL pode ser criada em Menu Do Admin->Conexões.

Data Source

Neste campo, é retornado uma ou mais queries SQL com os dados que serão utilizados na página. Este campo é opcional. É possível utilizar as variáveis {UserId} e {CurrentCulture} para filtrar registros.

Template

Utilizamos a linguagem Liquid para a renderização do HTML.

O que é o Liquid?

Para a renderização do HTML, utilizamos a linguagem Liquid.

No Infinity utilizamos a implementação em .NET criada por sebastienros junto com a comunidade, que se chama Fluid. Ela tem como vantagem em relação ao Liquid "tradicional" funcionalidades a mais como funções e foi desenvolvida com performance em mente. É 30% mais rápida que outras implementações de acordo com os benchmarks.

No JJPages, as seguintes variáveis estão disponíveis para uso:

Variável Descrição
DataSource Resultado da query SQL. Exemplo: {{ DataSource[0][0]["Foo"] }}. O primeiro indexador significa o primeiro resultado da query. O segundo, a primeira linha da query, e por último, qual a coluna. É possível consumir resultados e linhas dentro de loops for.
CurrentCulture Cultura atual da página. Exemplo: pt-BR.
UserId ID do usuário logado.
LegacyId ID legado do usuário logado.
AccountId ID autoincremento do usuário logado.
FirstName Primeiro nome do usuário logado.
LastName Último nome do usuário logado.
Name Nome completo do usuário logado.
UserEmail Email do usuário logado.
IsAdmin Indica se o usuário logado é administrador (true ou false).
Tip

Utilizando o atalho CTRL+Espaço, o editor de texto sugere automaticamente as variáveis.

Além das funções padrão disponíveis em nosso tutorial, as seguintes funções estão disponíveis:

Função Descrição
includePage Renderiza uma outra JJPage dentro da página atual. Funciona como uma espécie de partial view do ASP.NET Core MVC. Exemplo: {{ includePage("Agendas da Semana") }}.
isInRole Retorna um booleano caso o usuário atual esteja dentro ou não das roles Exemplo: {{ isInRole("Coordenador","Admin") }}.

Encode do HTML

Por padrão, em todo valor renderizado é feito o encode do HTML para prevenir ataques XSS. Para renderizar como HTML, é necessário utilizar o filtro raw.

{{ DataSource[0][0]["MeuHtml"] | raw }}

Filtros

Para utilizar filtros, utilize a variável {{Request["NomeDoCampo]}} onde nome do campo pode ser um valor do formulário ou da QueryString. É possível utilizar a variável request tanto no banco de dados quanto em seu template HTML. Não é possível utilizar filtros no modo iframe.

Exemplo:

select {Request["Nome"]} as NomeDoBanco
Tip

Na primeira renderização da página, o valor do formulário será retornado como NULL, a menos que tenha sido fornecido por meio da query string.

<form method="POST">
    {% assign formName = Request["Nome"] %} {% comment %} Também é possível utilizar Request.Nome {% endcomment %} 
    {% assign dbName = DataSource[0][0]["NomeDoBanco"] %} 
    {% assign nomes = "Alice,Bob,Charlie,David" | split: ","%}
    <h3>Nome do Formulário {{ formName }}</h3>
    <h3>Nome no Banco De Dados: {{ dbName }}</h3>
    <select name="Nome" onchange="this.form.submit()">
        {% for option in nomes %}
            <option value="{{ option }}"  {%if formName == option %} selected {%endif%}>{{ option }}</option>
        {% endfor %}
    </select>
</form>
Warning

Não se esqueça de configurar seu formulário como POST.
