JJReport é um produto da JJ Consulting para geração de relatórios SQL em tempo de execução.

Editor

No editor é possível escrever a query SQL do relatório. Filtros podem ser passados entre chaves ({}). É possível recuperar o usuário logado utilizando a keyword{USERID}. Em tempo de execução os filtros entre chaves são trocados para parâmetros do banco de dados para prevenir SQL injection. No SQL Server, {USERID} se torna @USERID por exemplo.

Tabelas

Opções para cada tabela retornada pelo relatório. Essas opções só podem ser configuradas executando e salvando o relatório ao menos 1 vez. É possível configurar paginação, opções de layout e verificar os campos de cada tabela retornada.

Filtros

Utilizando o JJMasterData, é possível renderizar um formulário com os filtros. É possível indicar se um filtro é obrigatório, caso um filtro seja obrigatório, o relatório não é executado sem este filtro.

Variáveis dos filtros

Variável Descrição
UserId Id do usuário logado
LegacyId Id legado do usuário logado
AccountId Id autoincremento do usuário logado
UserEmail Email do usuário logado

Configurações

Nessa tela é possível configurar as opções que se aplicam a todas tabelas do relatório.

Conexão Selecionada

Conexão selecionada para o relatório. É possível cadastrar novas conexões em [Painel do Admin->Conexões]

Executar ao Abrir

Essa opção indica se o relatório será executado imediatamente ao abrir ou apenas ao clicar em "Gerar Relatório".

Cache

Caso essa opção é habilitada, quando o relatório é executado sem filtros, é realizado o cache do relatório. A duração do cache só pode ser entre 1 minuto e 2 horas. O {USERID} não é considerado um filtro para o cache.
